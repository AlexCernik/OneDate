Introduction

This documentation will help you get familiar with the recurces of the One Piece API and show you now to make different queries, so that you can get the mostout of it

REST

Base url: https://apionepiece.herokuapp.com/api/

The base url contains information about all available API's resources. All reuqests are GET requests and go over https . All response will return data in Json.

Sample request

          https://apionepiece.herokuapp.com/api/
        
          
          {
            "characters":"https://apionepiece.herokuapp.com/api/characters",
          }

There are currently one available resources:

  • Characters: used to get all the character's.

Info and Pagination

The API will automatically paginate the responses. You will receive up to 15 documents per page.

Each resources contains an info object with information about the response.

Key Type Description
count int The length of the response.
nex string (url) Link to next page (if it exits)
prev string (url) Link to prev page (if it exits)

Character

There is a total of {{characters|length}} for the moments, character's sorted by id.

Character Scheme

Key Type Description
id int The id of character.
name string The name of the character.
image string Link to the character's image. All images are 300x500px and most are medium shots or portraits since they are intended to be used as avatars.
age int Age of the character's.
gender string The gender of the character('Female','Male' or 'Unknown').
reward string The reward of character.
fruit_name string Character fruit name.
fruit_type string Character fruit type.
race string The type or subspecies of the character ('Human','Cyborg' or 'Gyoji').
rank string Character rank type.
status string The status of the character ('Alive', 'Dead' or 'Unknown').
origin string Character origin.
occupation string The occupation of the character ('Capitan', 'Samurai' or 'Unknown').
created_on string Time at which the character was created in the database.
update_on string Time at whichthe character was update in the database.
